When downloading the distribution then take both the source and the macro components, these should NOT be mixed with a previous release. Users upgrading from 2001 (or older) versions must also download the spelling dictionaries as this release is no longer compatible with the older dictionaries.

Files supplied in the UNIX tar.gz format may be downloaded to Microsoft Windows/DOS environments, use Winzip or native UNIX tools to unpack the files.

Building from source

    If your system does not have a binary image then you may build one from the source. For UNIX a simple ./build from the source directory will build an executable image for a supported target. A build.bat and Microsoft Developer project are provided for Microsoft environments.

Upgrading from the January 02 Release

    IMPORTANT: Refer to the notes at the bottom of this page or upgrade0212.txt.

Quick Download Guide

    If you want to install JASSPA's MicroEmacs then the minimum executable download set is a complete installation or the following separate components:-

    • Executable image for your target.
    • memacros.xxx macro files that accompany the executable image.
    • Optionally, meehf.xxx the on-line help file (recommended).
    • Other components such as help files, icon's etc. may be optionally downloaded.

    Use NanoEmacs for a minimal download. Pre-built executables may be simply downloaded and run. NanoEmacs does not require any setup or additional files in order to operate.

Upgrading from January 02

    The following are notes on upgrading to the December 02 release from the January 02 release.

General Notes - all platforms

    If you are upgrading from the January 02 release then some of the binary data in the registry file <user>.erf file will have changed. Delete your existing <user>.erf file and re-configure using M-x user-setup (or via the menu Help->User Setup). Failure to do so will mean that some of the buffer modes are incorrect.

    All references to screen should be changed to frame i.e.

             82 change-screen-width
         50 change-screen-depth

    become

             change-frame-width 82
         change-frame-depth 50

    typically used in user start up scripts to change the screen size.

UNIX upgrade

    No special procedure above general.

Windows Upgrade

    This release currently does not include an InstallShield installation. This will be completed in the near future. To upgrade an existing January '02 Install shield installation then the following procedure may be used.

    JASSPA TAKES NO RESPONSIBILITY FOR ANY DAMAGE CAUSED TO ANY SYSTEM BY FOLLOWING THESE NOTES. RESPONSIBILITY LIES WHOLLY WITH THE USER IF YOU ARE UNSURE OF WHAT YOU ARE DOING THEN WAIT FOR THE INSTALL SHIELD IMAGE.

    Having denied all responsibility, we have been using this technique for years to upgrade the installed release without problems:-

    • Over-write the existing me32.exe image in c:/Program Files/Jasspa/MicroEmacs with the new image. DO NOT under any circumstances rename or move this image as may affect the Windows registry settings. If you want to make a backup then copy the image to a backup area before over-writing the existing executable image with the new image - you can restore this later if required.
    • Delete all files in c:/Program Files/Jasspa/MicroEmacs/macros to leave an empty directory (or move to the backup area). We assume that the user has not modified the files in this directory. If changes have been made then save them before deletion.
    • Unzip memacros.zip and meehf.zip into the empty macros directory.
    • Install any other new documentation files (i.e. winhelp etc.).
    • The <user>.erf files should be deleted (optionally backed up). On running me. M-x user-setup (or via the menu Help->User Setup) should be executed to restore the existing user settings.
